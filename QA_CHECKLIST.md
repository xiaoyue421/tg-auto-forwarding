# QA Checklist

## 启动与基础

- [ ] `python -m tg_forwarder web --config .env` 可正常启动
- [ ] 控制台可登录，`/api/status` 返回正常
- [ ] 控制台保存配置后可重新加载并显示一致

## 规则与转发

- [ ] 规则分组字段可保存并回显
- [ ] 优先级可保存并回显
- [ ] 分组启用/停用生效
- [ ] 分组折叠/展开正常
- [ ] 分组筛选与“在当前分组新增”正常
- [ ] “按优先级排序”后顺序符合预期

## 失败队列

- [ ] 失败任务可重试
- [ ] 不可重试错误会被跳过并有统计
- [ ] FloodWait 冷却中的任务会被跳过并有统计

## HDHive 签到

- [ ] 开启自动签到后，配置缺失会在保存阶段报错
- [ ] 网络错误可触发重试（观察 `next_retry_epoch`）
- [ ] 成功后当日不再重复自动签到
- [ ] 代理配置异常可在健康状态看到提示

## 健康与诊断

- [ ] `GET /api/health` 返回服务、队列、日志、签到状态
- [ ] `GET /api/v1/health` 返回与 `/api/health` 一致
- [ ] 控制台“导出诊断包”可下载 JSON 文件
- [ ] 诊断包中的敏感字段已脱敏（token/cookie/password 等）

## 日志

- [ ] 命中日志包含“命中详情”
- [ ] 未命中日志包含“未命中详情”
- [ ] 开启 `TG_FILE_LOG_ENABLED=true` 后生成按天滚动日志文件
